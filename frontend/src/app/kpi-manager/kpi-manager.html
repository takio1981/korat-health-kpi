ระบบจัดการข้อมูล KPI ↔ HDC

รวมการเปรียบเทียบชื่อตัวชี้วัด, โครงสร้างตาราง และการส่งออก/ส่งเข้า HDC ไว้ในขั้นตอนเดียว

ขั้นตอนการทำงาน (Workflow)

1

เทียบชื่อตัวชี้วัด

เปรียบเทียบ ชื่อตัวชี้วัด ระหว่าง HDC reports กับ Local kpi_indicators โดยใช้ table_process เป็น key — Sync ชื่อใหม่/อัปเดตชื่อเข้า Local

ผลที่ได้: รายการตัวชี้วัดใน Local ตรงกับ HDC

2

ส่งออกข้อมูล KPI ↔ HDC

รวม 2 phase: เตรียมโครงสร้างตาราง (Schema) + ส่งออกข้อมูล + ส่งเข้า HDC + ตารางเวลาอัตโนมัติ

Phase A: เทียบโครงสร้าง — สร้าง/sync schema (ทำครั้งเดียวตอน setup)
Phase B: ตรวจสอบ → Export → Sync HDC + Schedule (ทำประจำ)

เปรียบเทียบ Phase A vs Phase B

หัวข้อ Phase A — Schema Phase B — Data
ทำงานกับโครงสร้างตาราง (columns)ข้อมูลในตาราง (rows)
ทิศทาง2 ทาง (HDC ↔ Local)1 ทาง (Local → HDC)
ความถี่ครั้งเดียวตอน setup / เมื่อ schema เปลี่ยนรายเดือน (manual หรือ schedule)
เครื่องมือCREATE / ALTER TABLE + Sync rows ตาม directionตรวจสอบ → Export → Sync HDC → Scheduler
เริ่มจาก"เปรียบเทียบ Structure""ตรวจสอบข้อมูล" → "ส่งออกข้อมูล"

เคล็ดลับการใช้งาน

  • ครั้งแรก: ทำตามลำดับ ขั้น 1 → ขั้น 2 (Phase A) → ขั้น 2 (Phase B)
  • หลังจากนั้นใช้งานประจำ: ข้ามไปขั้น 2 → Phase B ได้เลย (ไม่ต้องทำ Phase A ซ้ำ)
  • ถ้าต้องการ สร้างฟอร์มจาก HDC columns ให้กดในขั้น 2 → Phase A ที่ตารางที่มีใน HDC
  • Schedule อัตโนมัติ ตั้งใน Phase B — ระบบ export + sync HDC ตามวันเวลาที่กำหนด
