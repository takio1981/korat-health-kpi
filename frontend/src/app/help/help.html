กัน login ซ้อนจากหลายเครื่อง (Single Session)

ระบบอนุญาตให้ใช้ 1 บัญชี = 1 อุปกรณ์ ในเวลาเดียวกันเท่านั้น

  • ถ้ามีคน login บัญชีเดิมจากเครื่องอื่นอยู่ + ใช้งานล่าสุด ≤ 5 นาที → ถูก block (HTTP 409)
  • ถ้าเจ้าของไม่ได้ใช้งานเกิน 5 นาที → login ใหม่ได้ (session เก่าจะถูก invalidate)
  • ถ้ามีคน login บัญชีเดิมจากเครื่องใหม่ → เครื่องเดิมจะ logout อัตโนมัติภายใน ~30 วินาที
  • กรณีฉุกเฉิน super_admin บังคับ logout user ได้ผ่านหน้า "จัดการผู้ใช้งาน"

เข้าสู่ระบบด้วย ThaID / ProviderID (รอเปิดใช้งาน)

บัญชี Thai Digital ID — ลงทะเบียนอัตโนมัติด้วยข้อมูลจากบัตรประชาชน

บัญชี Provider ID ของกระทรวงสาธารณสุข สำหรับบุคลากรในระบบ HDC/MOPH

ขณะนี้อยู่ระหว่างขอใช้บริการกับหน่วยงานเจ้าของ — กดปุ่มแล้วจะแจ้ง "รอเปิดใช้งาน" ให้ใช้ Username/Password ปกติไปก่อน

ตัวกรองสอดคล้องกัน (Cascading Filters)

  • เลือก อำเภอ → หน่วยบริการแสดงเฉพาะในอำเภอนั้น

  • เลือก หน่วยงาน → หมวดหมู่ + ตัวชี้วัด แสดงเฉพาะหน่วยงานนั้น

9.5 ตัวชี้วัดย่อย (Sub-Indicators)

ตัวชี้วัด 1 ตัว สามารถมีย่อยได้หลายตัว + บันทึกผลงานรายเดือนต่อ hospcode

เกณฑ์การประเมิน — ประเภทหน่วยบริการที่รับผิดชอบ

กำหนดว่าตัวชี้วัดใช้กับหน่วยบริการประเภทไหน (รพ. / รพ.สต. / สสอ. / สสจ.) ส่งผลต่อการกรอง "ตัวชี้วัดของ" ในหน้า dashboard

  1. เข้าหน้า "จัดการตัวชี้วัด" → Tab "ตัวชี้วัด" → กดแก้ไข/เพิ่ม
  2. ในกรอบ "เกณฑ์การประเมิน" (teal) เลือกโหมด:
    • any_one (เฉพาะบางประเภท) — ตัวชี้วัดใช้เฉพาะประเภทที่ติ๊กใน checkbox → เก็บใน required_off_types เป็น JSON array
    • all_required (ทุกประเภทบังคับ) — ตัวชี้วัดครอบคลุมทุกหน่วยบริการในขอบเขต → required_off_types = NULL
  3. กรณี any_one → ติ๊กประเภท (05=รพ.ศ., 06=รพ.ท., 07=รพ.ชุมชน, 10=ศูนย์ฯ, 18=รพ.สต., 02=สสอ., 01=สสจ., ...)
  4. กด "บันทึก" → ระบบเก็บเป็น JSON เช่น ["05","06","07"]
ตัวอย่าง:
  • ตัวชี้วัดโรงพยาบาล: any_one + ["05","06","07"]
  • ตัวชี้วัด รพ.สต.: any_one + ["18"]
  • ตัวชี้วัดระดับอำเภอ (สสอ.): any_one + ["02"]
  • ตัวชี้วัดที่ทุกหน่วยต้องรายงาน: all_required

  4. กรอกค่าเป้าหมาย + ผลงานรายเดือน → กด "บันทึกทั้งหมด"
  5. ระบบ upsert เฉพาะเซลล์ที่เปลี่ยน (diff detection) → ไม่ยิง API ซ้ำ
ผลงาน = ค่าเดือนล่าสุดที่คีย์ (ก.ย.→ต.ค.) | % = ผลงาน/เป้าหมาย × 100 (สีตามเกณฑ์ ≥80 เขียว / 50-79 เหลือง / <50 แดง)

คำนวณจากตัวชี้วัดย่อย (แถวหลัก)

เมื่อตัวชี้วัดมี sub-indicators → แถว main จะแสดงค่า เฉลี่ย จาก subs ทั้งหมด (AVG หารด้วยจำนวน sub)

• เป้าหมาย = AVG(target_value) ของ subs
• ต.ค.-ก.ย. = AVG(actual_value) รายเดือน
• ผลงานรวม = เดือนล่าสุดที่มีค่า
• % = ผลงาน/เป้าหมาย × 100

Format: จำนวนเต็มแสดงไม่มีทศนิยม (เช่น 70) | มีเศษแสดง 2 ตำแหน่ง (เช่น 70.50, 35.33)

ลบกระทู้

  • ตอบกลับทุกข้อความ → Telegram + Email + Notification ถึงเจ้าของกระทู้ + super_admin

13. คำถามที่พบบ่อย (FAQ)

ถาม: ลงทะเบียนแล้วเข้าใช้งานไม่ได้?

ตอบ: ต้องรอผู้ดูแลระบบอนุมัติก่อน จะได้รับแจ้งทาง Email เมื่ออนุมัติแล้ว

ถาม: ถูกปฏิเสธแล้วทำอย่างไร?

ตอบ: สมัครใหม่ด้วยเลขบัตรประชาชนเดิมได้เลย ระบบจะลบ account เก่าให้อัตโนมัติ

ถาม: ลืมรหัสผ่าน?

ตอบ: กดปุ่ม "ลืมรหัสผ่าน?" ในหน้า Login → กรอก Username → รับรหัสชั่วคราว 6 หลักทาง Email → ใช้ Login แล้วเปลี่ยนรหัสใหม่ | ถ้าไม่มี Email ลงทะเบียนไว้ ให้ติดต่อผู้ดูแลระบบ

ถาม: กดแก้ไขข้อมูลไม่ได้ ขึ้น "ระบบปิดการคีย์"?

ตอบ: ผู้ดูแลระบบ (super_admin) ล็อคการคีย์ไว้ ให้ติดต่อผู้ดูแลเพื่อปลดล็อค

ถาม: ข้อมูลที่กรอกเป็นข้อความ (ไม่ใช่ตัวเลข) คำนวณ % อย่างไร?

ตอบ: ถ้าเป้าหมายเป็นข้อความ เช่น "ผ่าน" และผลงานตรงกัน → 100% | ไม่ตรง → 0% | ถ้าเป็นตัวเลข → คำนวณปกติ

ถาม: ระบบออกจากระบบเอง?

ตอบ: ระบบมี Auto Logout เมื่อไม่ใช้งานเกินเวลาที่กำหนด เปิด browser/tab ใหม่จะ login ใหม่เมื่อ token หมดอายุ

ถาม: เปิด Dark Mode ได้อย่างไร?

ตอบ: กดปุ่ม 🌙 ที่ Header bar (หรือมุมขวาบนในหน้า Login) ระบบจำค่าไว้ กลับมาก็ยัง Dark อยู่

ถาม: สำรองข้อมูลได้อย่างไร?

ตอบ: super_admin เข้า "ตั้งค่าระบบ" → กดปุ่ม "สำรองฐานข้อมูลทั้งหมด" → download ไฟล์ JSON ครบทุกตาราง

ถาม: Admin รีเซ็ตรหัสผ่านให้ user ได้ไหม?

ตอบ: ได้ กดปุ่ม 🔑 ในหน้าจัดการผู้ใช้ → ระบบสร้างรหัสชั่วคราว 6 หลัก + ส่ง Email แจ้ง user → user login แล้วเปลี่ยนรหัสใหม่

ถาม: ตั้งค่า Telegram แจ้งเตือนอย่างไร?

ตอบ: สร้าง Bot ผ่าน @BotFather → เพิ่มเข้า Group → หา Chat ID → กรอกใน "ตั้งค่าระบบ" → กดทดสอบ → บันทึก

ถาม: แจ้งเตือนผู้สมัครใหม่ไปที่ไหนบ้าง?

ตอบ: 3 ช่องทางพร้อมกัน — Telegram Group, Email Admin, แจ้งเตือนในระบบ (super_admin) ตั้งค่าแต่ละช่องทางได้ในหน้า "ตั้งค่าระบบ"

ถาม: Admin สร้างผู้ใช้ใหม่ต้องรอ super_admin อนุมัติหรือไม่?

ตอบ: ใช่ ผู้ใช้ที่ถูกสร้างโดย admin อื่นๆ (ไม่ใช่ super_admin) จะอยู่สถานะ "รอการอนุมัติ" เฉพาะ super_admin สร้างจึงอนุมัติทันที

ถาม: ระบบขึ้นแจ้งว่า "ปิดการใช้งานชั่วคราว" ทำอย่างไร?

ตอบ: ผู้ดูแลระบบ (super_admin) เปิดโหมดปิดปรับปรุง กรุณารอจนกว่าจะเปิดให้ใช้งานอีกครั้ง

ถาม: แถบแดง "ทดสอบระบบ" คืออะไร?

ตอบ: แสดงเมื่อระบบอยู่ในช่วงทดสอบ เมื่อ Deploy จริงจะถูกนำออก

ถาม: "จัดการข้อมูล KPI" มี Tab อะไรบ้าง?

ตอบ: 4 Tab — 1) DB Compare (เปรียบเทียบ structure + Sync 2 ทิศทาง HDC↔Local) 2) สร้างแบบฟอร์ม (Form Builder) 3) Export ข้อมูล 4) Report Compare (เทียบ reports HDC กับ kpi_indicators)

ถาม: ทำไม Login แล้วได้รับ Email แจ้งเตือน?

ตอบ: ระบบส่ง Email แจ้งเตือนทุกครั้งที่ Login สำเร็จ (เวลา + IP) เพื่อความปลอดภัย หากไม่ใช่คุณ กรุณาเปลี่ยนรหัสผ่านทันที

ถาม: super_admin เปิด Dashboard แล้วไม่เห็นข้อมูล?

ตอบ: super_admin ไม่โหลดข้อมูลอัตโนมัติเพื่อลดภาระ กรุณาเลือกตัวกรอง (อำเภอ/หน่วยบริการ/หน่วยงาน) หรือกดปุ่ม "โหลดข้อมูลทั้งหมด"

ถาม: admin_ssj ไม่เห็นปุ่มลบ?

ตอบ: ปุ่มลบแสดงเฉพาะ super_admin เท่านั้น admin_ssj สามารถเพิ่ม/แก้ไข/เปิด-ปิดได้ แต่ไม่สามารถลบข้อมูลได้

ถาม: dropdown หน่วยบริการเรียงลำดับอย่างไร?

ตอบ: เรียงตามประเภทหน่วยบริการ (รพ. → สสอ. → รพ.สต. → ศูนย์สุขภาพ) แล้วตามชื่อ

ถาม: "ส่งข้อมูลเข้า HDC" ทำงานอย่างไร?

ตอบ: ระบบใช้ UPSERT (INSERT ... ON DUPLICATE KEY UPDATE) อัปเดตเฉพาะข้อมูลที่ key ตรงกัน ข้อมูลเดิมใน HDC ที่ไม่ซ้ำจะยังคงอยู่ ไม่ถูกลบ

ถาม: "ล้างข้อมูลผลงาน" ลบอะไรบ้าง?

ตอบ: ลบข้อมูลใน kpi_results + ข้อมูลในตาราง form_* ทั้งหมด (โครงสร้างตาราง/แบบฟอร์มยังอยู่) ต้องพิมพ์ "ยืนยันลบ" และควรสำรองข้อมูลก่อนทุกครั้ง

ถาม: สีข้อความใน "ข้อเสนอแนะ" คืออะไร?

ตอบ: แต่ละ username มีสีเฉพาะตัว เพื่อแยกว่าข้อความของใคร — super_admin = สีเขียวเข้มเสมอ (สงวนไว้) username อื่นจะได้สีจาก palette 10 สีอัตโนมัติ

ถาม: ตัวเลขที่เมนูข้อเสนอแนะหมายถึงอะไร?

ตอบ: จำนวนกระทู้ + ข้อความตอบกลับใหม่ที่ยังไม่ได้อ่าน เมื่อเข้าหน้าข้อเสนอแนะ ตัวเลขจะเคลียร์อัตโนมัติ
