## 1. ภาพรวมโปรเจค

### วัตถุประสงค์
ระบบ Korat Health KPI เป็นเว็บแอปพลิเคชันสำหรับติดตาม บันทึก และตรวจสอบผลงานตัวชี้วัดด้านสาธารณสุข ของหน่วยบริการในจังหวัดนครราชสีมา โดยรองรับปีงบประมาณไทย (ตุลาคม - กันยายน)

### เทคโนโลยีหลัก (Tech Stack)

| ส่วน | เทคโนโลยี | เวอร์ชัน |
|------|-----------|---------|
| **Frontend** | Angular (Standalone Components) | 21 |
| **Backend** | Express.js | 5 |
| **Database** | MySQL / MariaDB | 11.4+ |
| **UI Framework** | Tailwind CSS | 4 |
| **Charts** | ApexCharts | - |
| **Alerts** | SweetAlert2 | - |
| **Auth** | JWT (JSON Web Token) | - |
| **Container** | Docker + Docker Compose | - |
| **Web Server** | Nginx (Reverse Proxy + SSL) | Alpine |

### ระบบรักษาความปลอดภัย (Security)

| มาตรการ | รายละเอียด |
|---------|-----------|
| **JWT Token** | หมดอายุ 8 ชั่วโมง, เก็บ userId/username/deptId/role/hospcode |
| **Bcrypt** | เข้ารหัส password ด้วย salt 10 rounds |
| **Rate Limiting** | Login: 10 ครั้ง/15 นาที, API ทั่วไป: 300 ครั้ง/นาที |
| **Helmet.js** | ป้องกัน XSS, Clickjacking, MIME sniffing |
| **HTTPS/SSL** | Self-signed certificate (สามารถเปลี่ยนเป็น Let's Encrypt ได้) |
| **Idle Timeout** | ตั้งค่าได้ (ค่าเริ่มต้น 15 นาที) |

## 5. คุณสมบัติของระบบ

### 5.1 ระบบเข้าสู่ระบบ (Login)
- กรอก Username + Password
- ปุ่มแสดง/ซ่อนรหัสผ่าน (Eye Toggle)
- ป้องกัน Brute Force (จำกัด 10 ครั้ง/15 นาที)
- JWT Token อายุ 8 ชั่วโมง
- บันทึกประวัติ Login ทุกครั้ง (สำเร็จ/ล้มเหลว)
- Idle Timeout (หมดเวลาอัตโนมัติ ค่าเริ่มต้น 15 นาที)

### 5.2 ระบบลงทะเบียน (Register)
- เปิดให้ลงทะเบียนเองได้ (ไม่ต้อง Login)
- กรอกข้อมูล: ชื่อผู้ใช้, รหัสผ่าน, ชื่อ-นามสกุล, เบอร์โทร, อำเภอ, หน่วยบริการ, หน่วยงาน
- ตรวจสอบ Username ซ้ำ
- ตรวจสอบรหัสผ่านขั้นต่ำ 6 ตัวอักษร
- ตรวจสอบเบอร์โทร 10 หลัก (แสดงรูปแบบ XX-XXXX-XXXX)
- ยืนยันรหัสผ่าน (Confirm Password)
- บทบาทเริ่มต้นคือ `user`

### 5.3 ภาพรวมตัวชี้วัด (Dashboard)
- **ตารางข้อมูล**: แสดงตัวชี้วัดทั้งหมด พร้อมคอลัมน์ 12 เดือน (ต.ค. - ก.ย.)
- **แก้ไขรายเดือน**: คลิกเซลล์เพื่อแก้ไขผลงานรายเดือน
- **สถานะ**: แสดงสถานะ Pending (รออนุมัติ) / Approved (อนุมัติแล้ว)
- **กรอง**: ตามหมวดหมู่หลัก, ตัวชี้วัด, หน่วยงาน, ปีงบประมาณ, สถานะ, หน่วยบริการ, อำเภอ
- **ค้นหา**: ค้นหาข้อความอิสระ
- **Pagination**: แบ่งหน้า 20 รายการ/หน้า
- **สถิติแบบ Animate**: อัตราความสำเร็จ %, จำนวนที่บันทึก, จำนวนหน่วยงาน, รอดำเนินการ
- **กราฟแนวโน้ม**: แสดงกราฟเส้นรายเดือนของตัวชี้วัดที่เลือก

### 5.4 ระบบอนุมัติ/ตีกลับ KPI (Workflow)
- **อนุมัติ** (admin_ssj, super_admin): อนุมัติผลงาน KPI → ล็อคไม่ให้แก้ไข
- **ตีกลับ** (admin_ssj, super_admin): ตีกลับพร้อมระบุเหตุผลและเดือนที่ต้องแก้ไข
- **ตอบกลับ** (user): ตอบกลับเหตุผลที่ถูกตีกลับ
- **ปลดล็อค** (super_admin เท่านั้น): ปลดล็อค KPI ที่อนุมัติแล้วเพื่อแก้ไข
- **ประวัติการตีกลับ**: ดูประวัติการตีกลับ/ตอบกลับทั้งหมดของแต่ละตัวชี้วัด

### 5.5 รายงานสถิติและกราฟ (Charts)
- กราฟแท่ง (Bar Chart): เปรียบเทียบเป้าหมาย vs ผลงาน
- กราฟเส้น (Line Chart): แนวโน้มรายเดือน
- กราฟวงกลม (Pie Chart): สัดส่วนผลงาน
- กรองตามปี, หมวดหมู่หลัก, ตัวชี้วัด, หน่วยงาน
- แสดงสถิติแบบ Animate

### 5.6 รายงานสรุปผล (Reports)
- **4 รูปแบบรายงาน**:
  - รายข้อตัวชี้วัด
  - รายหน่วยบริการ
  - รายอำเภอ
  - รายปีงบประมาณ
- กรองตามปี, อำเภอ, หน่วยบริการ, หน่วยงาน
- สรุปจำนวนตัวชี้วัด, หน่วยบริการ, อัตราความสำเร็จ
- กราฟประกอบรายงาน

### 5.7 ระบบแจ้งเตือน (Notifications)
- แจ้งเตือน KPI รออนุมัติ
- แจ้งเตือนการตีกลับ
- แจ้งเตือนการตอบกลับ
- ไอคอนกระดิ่งพร้อมจำนวนที่ยังไม่อ่าน (Badge แดง + Pulse Animation)
- กรองตามประเภท: ทั้งหมด / อนุมัติ / ตีกลับ / ตอบกลับ
- ทำเครื่องหมายอ่านแล้ว

### 5.8 จัดการผู้ใช้งาน (User Management)
- **สร้างผู้ใช้ใหม่**: กำหนด username, password, ชื่อ-นามสกุล, เบอร์โทร, หน่วยบริการ, หน่วยงาน, บทบาท
- **แก้ไขผู้ใช้**: เปลี่ยนข้อมูลทุกฟิลด์ (รหัสผ่านเปลี่ยนเฉพาะเมื่อกรอก)
- **ลบผู้ใช้**: ลบพร้อมยืนยัน
- **รีเซ็ตรหัสผ่าน**: รีเซ็ตเป็น "password123"
- **กรอง/ค้นหา**: ตามบทบาท, หน่วยงาน, ข้อความ
- **Pagination**: แบ่งหน้า 10 รายการ/หน้า
- **จำกัดสิทธิ์**: admin_cup เห็นเฉพาะผู้ใช้ในหน่วยงานเดียวกัน

### 5.9 สร้าง KPI ปีงบประมาณใหม่ (KPI Setup)
- เลือกปีงบประมาณ, อำเภอ, หน่วยบริการ, หน่วยงาน
- โหลด Template ตัวชี้วัดจากระบบ
- กรอกเป้าหมายและค่ารายเดือน
- **2 โหมดบันทึก**:
  - เพิ่มทั้งหมด (เขียนทับ)
  - เพิ่มเฉพาะที่ยังไม่มี
- ตรวจสอบข้อมูลเดิมก่อนเขียนทับ
- **admin_ssj**: ล็อคหน่วยงานเป็นของตัวเอง (ทุก hospcode)
- **super_admin**: เลือกได้ทุกหน่วยงาน

### 5.10 จัดการตัวชี้วัด (KPI Manage) — Super Admin เท่านั้น
- **4 แท็บ**:
  - ตัวชี้วัดย่อย (Indicators)
  - ตัวชี้วัดหลัก (Main Indicators)
  - ยุทธศาสตร์ (Strategies / Main Yut)
  - หน่วยงาน (Departments)
- CRUD: สร้าง, แก้ไข, ลบ
- เปิด/ปิดการใช้งาน (Toggle Active)
- ค้นหาและกรอง

### 5.11 ประวัติการใช้งาน (Audit Logs) — Super Admin เท่านั้น
- แสดง Log ทุกการเปลี่ยนแปลงข้อมูล (INSERT, UPDATE, DELETE, APPROVE, REJECT, REPLY)
- แสดง username, เวลา, ประเภท, ตาราง, รายละเอียด, IP
- กรองตามประเภทการกระทำ
- ค้นหาข้อความ
- **สำรองข้อมูล**: Export เป็นไฟล์ CSV
- **ล้างข้อมูล**: ลบ Log ทั้งหมด (ต้องพิมพ์ "CONFIRM" ยืนยัน 2 ครั้ง)
- Pagination: 20 รายการ/หน้า

### 5.12 ตั้งค่าระบบ (Settings) — Super Admin เท่านั้น
- **Idle Timeout นาที**: ระยะเวลาก่อนหมดเวลา (ค่าเริ่มต้น 15 นาที)
- **Idle Timeout วินาที**: เพิ่มเติมจากนาที
- **Idle Countdown**: เวลานับถอยหลังก่อนออกจากระบบ (ค่าเริ่มต้น 30 วินาที)
- **Max Login Attempts**: จำนวนครั้งสูงสุดที่ Login ผิดได้ (ค่าเริ่มต้น 10 ครั้ง)
- **Log Retention Days**: จำนวนวันเก็บ Log (ค่าเริ่มต้น 90 วัน)
- **Auto Backup**: เปิด/ปิด Backup อัตโนมัติ
- **System Version**: เวอร์ชันระบบ

### 5.13 เปลี่ยนรหัสผ่าน (Change Password)
- ทุกผู้ใช้เปลี่ยนรหัสผ่านตนเองได้
- กรอก: รหัสผ่านปัจจุบัน, รหัสผ่านใหม่, ยืนยันรหัสผ่านใหม่
- ตรวจสอบรหัสผ่านขั้นต่ำ 6 ตัวอักษร
- อยู่ที่เมนูด้านล่าง Sidebar

## 6. คู่มือการใช้งาน

### 6.1 การเข้าสู่ระบบ

1. เปิดเว็บเบราว์เซอร์ ไปที่ `http://[server]:8881/khupskpi/`
2. กรอก **ชื่อผู้ใช้** และ **รหัสผ่าน**
3. กด **เข้าสู่ระบบ**
4. หากเข้าสู่ระบบสำเร็จ จะเข้าสู่หน้ารายงานสถิติ

> หากยังไม่มีบัญชี กดลิงก์ **"ยังไม่มีบัญชี? ลงทะเบียน"** ด้านล่างปุ่ม Login

### 6.2 การลงทะเบียนผู้ใช้ใหม่

1. กดลิงก์ **"ลงทะเบียน"** จากหน้า Login
2. กรอกข้อมูลทุกช่อง:
   - **ชื่อผู้ใช้**: ต้องไม่ซ้ำกับผู้ใช้อื่น
   - **รหัสผ่าน**: อย่างน้อย 6 ตัวอักษร (a-z, A-Z, 0-9, อักขระพิเศษ)
   - **ยืนยันรหัสผ่าน**: ต้องตรงกับรหัสผ่าน
   - **ชื่อ-นามสกุล**: ชื่อจริง-นามสกุล
   - **เบอร์โทร**: 10 หลัก (ระบบจะแสดงรูปแบบ XX-XXXX-XXXX)
   - **อำเภอ**: เลือกอำเภอ → ระบบจะกรองหน่วยบริการให้
   - **หน่วยบริการ**: เลือก รพ./รพ.สต.
   - **หน่วยงาน**: เลือกกลุ่มงาน (ถ้ามี)
3. กด **ลงทะเบียน**
4. รอ Admin อนุมัติ (บทบาทเริ่มต้นคือ `user`)

### 6.3 การบันทึกผลงาน KPI (Dashboard)

1. ไปที่เมนู **"ภาพรวมตัวชี้วัด"**
2. เลือกตัวกรอง (ปีงบประมาณ, หน่วยงาน, หมวดหมู่ ฯลฯ) ตามต้องการ
3. คลิกที่ **ไอคอนดินสอ (แก้ไข)** ของตัวชี้วัดที่ต้องการ
4. กรอกค่าผลงานรายเดือน (ต.ค. - ก.ย.)
5. กด **บันทึก**
6. สถานะจะเป็น **"Pending"** (รออนุมัติ)

> ตัวชี้วัดที่ถูก Lock แล้วจะไม่สามารถแก้ไขได้ (ต้องรอ Super Admin ปลดล็อค)

### 6.4 การอนุมัติ/ตีกลับ KPI (สำหรับ admin_ssj / super_admin)

**อนุมัติ:**
1. ไปที่ **"ภาพรวมตัวชี้วัด"**
2. ดูตัวชี้วัดที่มีสถานะ **"Pending"**
3. กดปุ่ม **"อนุมัติ"** (เลือกทีละรายการหรือหลายรายการ)
4. ยืนยันการอนุมัติ → ข้อมูลจะถูก Lock

**ตีกลับ:**
1. กดปุ่ม **"ตีกลับ"** ที่ตัวชี้วัดที่ต้องการ
2. เลือกเดือนที่ต้องแก้ไข
3. ระบุเหตุผลการตีกลับ
4. กด **ตีกลับ** → ผู้ใช้จะได้รับแจ้งเตือน

### 6.5 การสร้าง KPI ปีงบประมาณใหม่ (KPI Setup)

1. ไปที่เมนู **"+KPI ปีงบประมาณใหม่"**
2. เลือก **ปีงบประมาณ** (เช่น 2570)
3. เลือก **อำเภอ** → ระบบจะกรองหน่วยบริการ
4. เลือก **หน่วยบริการ** ที่ต้องการ
5. เลือก **หน่วยงาน** (admin_ssj จะถูกล็อคเป็นหน่วยงานตัวเอง)
6. ระบบจะโหลด Template ตัวชี้วัดทั้งหมดให้
7. กรอก **เป้าหมาย** และ **ค่ารายเดือน** (ถ้ามี)
8. กด **บันทึกข้อมูล**
9. เลือกโหมด:
   - **เพิ่มทั้งหมด (เขียนทับ)**: แทนที่ข้อมูลเดิมทั้งหมด
   - **เพิ่มเฉพาะที่ยังไม่มี**: เพิ่มเฉพาะตัวชี้วัดใหม่ ไม่แทนที่ข้อมูลเดิม

### 6.6 การจัดการผู้ใช้งาน (สำหรับ Admin)

1. ไปที่เมนู **"จัดการผู้ใช้งาน"**
2. **สร้างผู้ใช้ใหม่**: กดปุ่ม **"+ เพิ่มผู้ใช้งาน"**
   - กรอก: ชื่อผู้ใช้, รหัสผ่าน, ยืนยันรหัสผ่าน, ชื่อ-นามสกุล, เบอร์โทร
   - เลือก: อำเภอ → หน่วยบริการ, หน่วยงาน, บทบาท
   - กด **บันทึก**
3. **แก้ไข**: กดไอคอนดินสอ → แก้ไขข้อมูล → บันทึก
4. **ลบ**: กดไอคอนถังขยะ → ยืนยันการลบ
5. **รีเซ็ตรหัสผ่าน**: กดปุ่มรีเซ็ต → รหัสผ่านจะเปลี่ยนเป็น "password123"

> admin_cup จะเห็นและจัดการได้เฉพาะผู้ใช้ในหน่วยงาน (dept_id) เดียวกัน

### 6.7 การจัดการตัวชี้วัด (สำหรับ Super Admin)

1. ไปที่เมนู **"จัดการตัวชี้วัด"**
2. เลือกแท็บ: ตัวชี้วัดย่อย / ตัวชี้วัดหลัก / ยุทธศาสตร์ / หน่วยงาน
3. **เพิ่ม**: กดปุ่ม **"+ เพิ่ม"** → กรอกข้อมูล → บันทึก
4. **แก้ไข**: กดไอคอนดินสอ → แก้ไข → บันทึก
5. **ลบ**: กดไอคอนถังขยะ → ยืนยัน
6. **เปิด/ปิดใช้งาน**: กดสวิตช์ Toggle → สถานะเปลี่ยนทันที

### 6.8 การดูรายงาน

**รายงานสถิติ (Charts):**
1. ไปที่เมนู **"รายงานสถิติ"**
2. เลือกตัวกรอง (ปี, หมวดหมู่, ตัวชี้วัด, หน่วยงาน)
3. ดูกราฟแท่ง, กราฟเส้น, กราฟวงกลม

3. เลือกตัวกรอง
4. ดูตารางสรุป + กราฟประกอบ

### 6.9 การเปลี่ยนรหัสผ่าน

1. กด **"เปลี่ยนรหัสผ่าน"** ที่ด้านล่างเมนู Sidebar
2. กรอก **รหัสผ่านปัจจุบัน**
3. กรอก **รหัสผ่านใหม่** (อย่างน้อย 6 ตัวอักษร)
4. กรอก **ยืนยันรหัสผ่านใหม่**
5. กด **บันทึก**

### 6.10 การดูแจ้งเตือน

3. กรองตามประเภท: ทั้งหมด / อนุมัติ / ตีกลับ / ตอบกลับ
4. กด **"อ่านแล้ว"** เพื่อทำเครื่องหมาย

### 7.2 การติดตั้งแบบ Development (ไม่ใช้ Docker)

**ขั้นตอนที่ 1: สร้างฐานข้อมูล**
```bash
# เข้า MySQL
mysql -u root -p

# สร้าง Database
CREATE DATABASE khups_kpi_db CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

# Import Schema + Seed Data
mysql -u root -p khups_kpi_db < database/khups_kpi_db.sql
```

**ขั้นตอนที่ 2: ตั้งค่า Backend**
```bash
cd kpi-backend

# สร้างไฟล์ .env
cat > .env << EOF
PORT=8830
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=khups_kpi_db
SECRET_KEY=your_secret_key_here_change_this
EOF

# ติดตั้ง Dependencies
npm install

# เริ่มต้น Backend
node server.js
# → Server running on port 8830
```

**ขั้นตอนที่ 3: ตั้งค่า Frontend**
```bash
cd kpi-web

# ติดตั้ง Dependencies
npm install

# เริ่มต้น Development Server
npx ng serve --port 4200
# → http://localhost:4200
```

> **หมายเหตุ**: ต้องแก้ไข API URL ใน `kpi-web/src/app/services/auth.ts` ให้ชี้ไปที่ `http://localhost:8830/khupskpi/api`

### 7.3 การ Deploy ด้วย Docker

**ขั้นตอนที่ 1: สร้างไฟล์ .env**
```bash
# สร้างไฟล์ .env ที่ root ของโปรเจค
cat > .env << EOF
DB_HOST=your_db_host
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=khups_kpi_db
SECRET_KEY=your_very_strong_secret_key_change_this
EOF
```

**ขั้นตอนที่ 2: Build และ Start**
```bash
# Build + Start ทุก Service
docker-compose up -d --build

# ตรวจสอบสถานะ
docker-compose ps

# ดู Logs
docker-compose logs -f
```

**ขั้นตอนที่ 3: เข้าใช้งาน**
```
HTTP:  http://[server-ip]:8881/khupskpi/
HTTPS: https://[server-ip]:8843/khupskpi/
API:   http://[server-ip]:8881/khupskpi/api/status
```
