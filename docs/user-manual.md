## 1. ภาพรวมระบบ

### ระบบนี้คืออะไร?
ระบบบันทึกผลงาน KPI ด้านสุขภาพ สำหรับสำนักงานสาธารณสุขจังหวัดนครราชสีมา ใช้สำหรับ:
- บันทึกผลงานตัวชี้วัดด้านสุขภาพรายเดือน (ปีงบประมาณ ต.ค. - ก.ย.)
- ติดตามเป้าหมายและผลงาน แยกตามหน่วยบริการ/อำเภอ/หน่วยงาน
- อนุมัติ/ตรวจสอบผลงาน โดยผู้ดูแลระบบ
- รายงานสรุปผลเป็นกราฟและตาราง

### สิทธิ์ผู้ใช้งาน (Roles) — 10 ระดับ

| กลุ่ม | Role | คำอธิบาย | ขอบเขตข้อมูล |
|-------|------|---------|-------------|
| **ผู้ใช้งาน** | user_hos | ผู้ใช้ รพ. | เฉพาะ hospcode + dept ตัวเอง |
| | user_sso | ผู้ใช้ รพ.สต. | เฉพาะ hospcode + dept ตัวเอง |
| | user_cup | ผู้ใช้ CUP (อำเภอ) | เฉพาะ dept ตัวเอง + ทุก hospcode ในอำเภอ |
| | user_ssj | ผู้ใช้ สสจ. | เฉพาะ hospcode + dept ตัวเอง |
| **ผู้ดูแล** | admin_hos | ผู้ดูแล รพ. | ทุก dept เฉพาะ hospcode ตัวเอง |
| | admin_sso | ผู้ดูแล รพ.สต. | ทุก dept เฉพาะ hospcode ตัวเอง |
| | admin_cup | ผู้ดูแล CUP (อำเภอ) | ทุก dept + ทุก hospcode ในอำเภอ |
| | admin_ssj | ผู้ดูแล สสจ. | ทุก hospcode เฉพาะ dept ตัวเอง |
| **สูงสุด** | super_admin | ผู้ดูแลระบบสูงสุด | เห็นทั้งหมด ทำได้ทั้งหมด |

## 2. การลงทะเบียน

### ขั้นตอนการลงทะเบียนผู้ใช้ใหม่

```
ขั้นตอนที่ 1: เปิดหน้าลงทะเบียน
    ↓
ขั้นตอนที่ 2: กรอกข้อมูลให้ครบถ้วน
    ↓
ขั้นตอนที่ 3: กดปุ่ม "ลงทะเบียน"
    ↓
ขั้นตอนที่ 4: รอผู้ดูแลระบบอนุมัติ
    ↓
ขั้นตอนที่ 5: ได้รับแจ้งผลทาง Email (ถ้ากรอก Email ไว้)
    ↓
ขั้นตอนที่ 6: เข้าสู่ระบบด้วย Username + Password
```

### ข้อมูลที่ต้องกรอก

| ลำดับ | ช่อง | บังคับ | เงื่อนไข |
|-------|------|-------|---------|
| 1 | ชื่อผู้ใช้งาน (Username) | ✅ | ขั้นต่ำ 6 ตัวอักษร, a-z, A-Z, 0-9, อักขระพิเศษ |
| 2 | รหัสผ่าน | ✅ | ขั้นต่ำ 6 ตัวอักษร, ต้องมี a-z + A-Z + 0-9 + อักขระพิเศษ ครบทุกประเภท |
| 3 | ยืนยันรหัสผ่าน | ✅ | ต้องตรงกับรหัสผ่าน |
| 4 | ชื่อ | ✅ | — |
| 5 | นามสกุล | ✅ | — |
| 6 | อำเภอ | — | เลือกเพื่อกรองหน่วยบริการ |
| 7 | หน่วยบริการ | ✅ | เลือกจาก dropdown (กรองตามอำเภอ) |
| 8 | เบอร์โทรศัพท์ | ✅ | ตัวเลข 10 หลัก (ระบบจัดรูปแบบให้อัตโนมัติ) |
| 9 | อีเมล (Email) | — | ใช้รับแจ้งผลอนุมัติ/ไม่อนุมัติ ถ้าไม่กรอกจะไม่ส่ง |
| 10 | หน่วยงาน | ✅ | เลือกจาก dropdown |
| 11 | เลขบัตรประชาชน 13 หลัก | ✅ | ต้องผ่าน Check Digit (Modulus 11) + ไม่ซ้ำกับที่สมัครแล้ว |
| 12 | สิทธิ์ที่ต้องการ | ✅ | เลือก Role ที่ต้องการ (การอนุมัติขึ้นอยู่กับผู้ดูแล) |

### ระบบตรวจสอบความแข็งแรงของรหัสผ่าน
- 🔴 **อ่อน** — ยังไม่ครบเงื่อนไข
- 🟡 **ปานกลาง** — ครบเงื่อนไขพื้นฐาน
- 🟢 **แข็งแรง** — ครบทุกเงื่อนไข + ยาว 10 ตัวขึ้นไป

### เมื่อถูกปฏิเสธ
- ได้รับแจ้งทาง Email พร้อมเหตุผล
- สามารถ **สมัครใหม่** ด้วยเลขบัตรประชาชนเดิมได้ (ระบบจะลบ account เก่าอัตโนมัติ)

## 3. การเข้าสู่ระบบ

### ขั้นตอน

```
ขั้นตอนที่ 1: เปิดระบบ (เข้า URL ของระบบ)
    → ระบบจะเข้าหน้า Login ทุกครั้งที่เปิด browser ใหม่
    ↓
ขั้นตอนที่ 2: กรอก Username
    ↓
ขั้นตอนที่ 3: กรอก Password (กดไอคอนตาเพื่อดูรหัสผ่าน)
    ↓
ขั้นตอนที่ 4: กดปุ่ม "เข้าสู่ระบบ"
```

### สถานะที่อาจพบ
| สถานะ | ข้อความ |
|-------|--------|
| ✅ สำเร็จ | เข้าสู่หน้า Dashboard |
| ⏳ รอการอนุมัติ | "บัญชีของคุณยังรอการอนุมัติ" |
| ❌ ถูกปฏิเสธ | "คำขอลงทะเบียนถูกปฏิเสธ" |
| 🔒 ถูกปิดใช้งาน | "บัญชีนี้ถูกปิดใช้งาน" |
| ⚠️ รหัสผ่านผิด | "ชื่อผู้ใช้งานหรือรหัสผ่านไม่ถูกต้อง" |
| 🚫 ล็อค 15 นาที | Login ผิดเกินจำนวนครั้งที่กำหนด |

### ระบบ Auto Logout
- ไม่มีการใช้งานเกินเวลาที่กำหนด → แสดงหน้าต่างเตือน + นับถอยหลัง
- กด "ยกเลิก" → ใช้งานต่อ
- หมดเวลา หรือ กด "ยืนยัน" → ออกจากระบบ

### ลืมรหัสผ่าน

```
ขั้นตอนที่ 1: กดปุ่ม "ลืมรหัสผ่าน?" ในหน้า Login
    ↓
ขั้นตอนที่ 2: กรอก Username ของคุณ
    ↓
ขั้นตอนที่ 3: กดปุ่ม "ส่งรหัสชั่วคราว"
    → ระบบส่งรหัส 6 หลักไปยัง Email ที่ลงทะเบียนไว้
    ↓
ขั้นตอนที่ 4: เปิด Email → คัดลอกรหัส 6 หลัก
    ↓
ขั้นตอนที่ 5: นำรหัส 6 หลักมาใส่ในช่อง Password → กดเข้าสู่ระบบ
    ↓
ขั้นตอนที่ 6: ระบบบังคับให้เปลี่ยนรหัสผ่านใหม่ทันที
    → มี Strength bar + Checklist เงื่อนไข + ปุ่มดวงตาเปิด/ปิดดูรหัส
    ↓
ขั้นตอนที่ 7: เปลี่ยนสำเร็จ → กลับหน้า Login → เข้าด้วยรหัสใหม่
```

**หมายเหตุ:**
- รหัสชั่วคราว **หมดอายุใน 15 นาที**
- ถ้าไม่มี Email ลงทะเบียนไว้ → ไม่สามารถรีเซ็ตเองได้ ต้องติดต่อผู้ดูแลระบบ
- เปลี่ยนรหัสผ่านใหม่ต้องครบเงื่อนไข: a-z, A-Z, 0-9, อักขระพิเศษ อย่างน้อยประเภทละ 1 ตัว

### เกี่ยวกับระบบ (ปุ่มในหน้า Login)
กดปุ่ม **"เกี่ยวกับระบบและวัตถุประสงค์"** เพื่อดู:
- วัตถุประสงค์ 5 ข้อ
- ความจำเป็นที่ต้องพัฒนาระบบ (4 ด้าน)
- ขอบเขตของระบบ

## 4. หน้าแรก — กราฟและสถิติ

### เมนูด้านซ้าย (Sidebar)
เมนูที่แสดงขึ้นอยู่กับสิทธิ์ของผู้ใช้

| เมนู | ไอคอน | ใครเห็น |
|------|-------|--------|
| รายงานสถิติ | 📊 | ทุกคน |
| รายงานสรุปผล | 📄 | ทุกคน |
| แจ้งเตือน 🔔 | 🔔 | ทุกคน (สั่นเมื่อมีแจ้งเตือนใหม่) |
| ภาพรวมตัวชี้วัด | 📋 | ทุกคน |
| จัดการผู้ใช้งาน | 👥 | Admin ทุกระดับ |
| จัดการตัวชี้วัด | ✅ | super_admin เท่านั้น |
| ประวัติการใช้งาน | 🕐 | super_admin เท่านั้น |
| +KPI ปีงบประมาณใหม่ | ➕ | admin_ssj + super_admin |
| Export ข้อมูล KPI | 📤 | super_admin เท่านั้น |
| สร้างแบบฟอร์ม KPI | 📝 | super_admin เท่านั้น |
| ตั้งค่าระบบ | ⚙️ | super_admin เท่านั้น |
| คู่มือการใช้งาน | 📖 | ทุกคน |

### กราฟที่แสดง
1. **กราฟแท่ง** — เปรียบเทียบเป้าหมาย VS ผลงานรวม (แยกตามหมวดหมู่)
2. **กราฟเส้น** — แนวโน้มผลงานรายเดือน (ต.ค. - ก.ย.) + เส้นค่าเฉลี่ยเป้าหมาย
3. **แผนที่อำเภอ** — ผลงาน KPI รายอำเภอ สี 3 ระดับ (เขียว ≥80% / เหลือง 50-79% / แดง <50%)
4. **กราฟวงกลม** — สัดส่วนผลงานแยกตามหน่วยงาน

### แผนที่ผลงาน KPI รายอำเภอ
- แสดง Grid card แต่ละอำเภอ พร้อมวงกลมเปอร์เซ็นต์
- สี 3 ระดับ:
  - 🟢 เขียว (≥80%) = บรรลุเป้าหมาย
  - 🟡 เหลือง (50-79%) = ปานกลาง
  - 🔴 แดง (<50%) = ต้องปรับปรุง
- แสดงสรุปจำนวนอำเภอในแต่ละระดับ
- เปลี่ยนตามตัวกรองที่เลือก (หมวดหมู่/ตัวชี้วัด/หน่วยงาน/ปี)

### ตัวกรอง
- หมวดหมู่หลัก
- ตัวชี้วัด
- หน่วยงาน
- ปีงบประมาณ

## 5. หน้า Dashboard — ภาพรวมตัวชี้วัด

### ส่วนประกอบหลัก

#### 5.1 แถบปุ่มด้านบน

| ปุ่ม | สี | ใครเห็น | ฟังก์ชัน |
|-----|-----|--------|---------|
| ล้างตัวกรอง | เทา | ทุกคน | ล้าง filter ทั้งหมด |
| เพิ่ม | เขียว | ทุกคน (ถ้าไม่ถูกล็อค) | เปิดหน้าต่างเพิ่มตัวชี้วัด |
| แก้ไข | เหลือง | ทุกคน (ถ้าไม่ถูกล็อค) | เปิดโหมดแก้ไขตาราง |
| 🔒 ระบบปิดการคีย์ | แดง | User ที่ถูกล็อค | แสดงเหตุผลล็อค |
| อนุมัติทั้งหมด | น้ำเงิน | admin_ssj + super_admin | อนุมัติ KPI ที่รอตรวจสอบ |
| ตีกลับทั้งหมด | แดง | admin_ssj + super_admin | ส่งคืน KPI ทั้งหมด |

#### 5.2 ตารางข้อมูล

| คอลัมน์ | คำอธิบาย |
|---------|---------|
| หมวดหมู่ | ชื่อหมวดหมู่หลัก |
| ตัวชี้วัด | ชื่อ KPI + ปุ่มจัดการ |
| หน่วยงาน | ชื่อหน่วยงาน |
| ปีงบฯ | ปีงบประมาณ |
| หน่วยบริการ | ชื่อ รพ./รพ.สต. |
| อำเภอ | ชื่ออำเภอ |
| เป้าหมาย | ค่าเป้าหมาย (ตัวเลข/ข้อความ) |
| ผลงานรวม | ค่าล่าสุดที่บันทึก |
| % | เปอร์เซ็นต์ (ถ้าข้อความเหมือนกัน = 100%) |
| ต.ค. - ก.ย. | ข้อมูลรายเดือน 12 เดือน |

#### 5.3 ปุ่มจัดการรายตัวชี้วัด (คอลัมน์ตัวชี้วัด)

| ปุ่ม | ไอคอน | ใครเห็น | ฟังก์ชัน |
|-----|-------|--------|---------|
| อนุมัติ | ✅ | admin_ssj/super_admin (สถานะ Pending) | อนุมัติข้อมูล |
| ตีกลับ | ↩️ | admin_ssj/super_admin (สถานะ Pending) | ส่งคืนแก้ไข |
| ปลดล็อค | 🔓 | super_admin (สถานะ Approved) | ปลดล็อคเป้าหมาย |
| ยื่นอุทธรณ์ | ⚖️ | User + admin พื้นที่ (สถานะ Approved + เปิดอุทธรณ์) | ยื่นอุทธรณ์ |
| อนุมัติอุทธรณ์ | ✅ | admin_ssj/super_admin (สถานะ Appeal) | อนุมัติอุทธรณ์ |
| ปฏิเสธอุทธรณ์ | ❌ | admin_ssj/super_admin (สถานะ Appeal) | ปฏิเสธอุทธรณ์ |
| กรอกแบบฟอร์ม | 📋 | ทุกคน (ถ้ามี form schema) | เปิด Dynamic Form |
| ดูแนวโน้ม | 📈 | ทุกคน | แสดงกราฟเส้น + เส้นเป้าหมายปะ |

#### 5.4 โหมดแก้ไข
1. กดปุ่ม **"แก้ไข"** → ตารางเปลี่ยนเป็น input fields
2. แก้ไขค่าเป้าหมายหรือคะแนนรายเดือน (ตัวเลข/ข้อความ)
3. ช่องที่แก้ไขจะ **เปลี่ยนสีส้ม** เพื่อแสดงว่ามีการเปลี่ยนแปลง
4. กดปุ่ม **"บันทึก"** → ยืนยัน → บันทึก
5. กดปุ่ม **"คืนค่า"** → กลับไปค่าเดิม
6. กดปุ่ม **"ยกเลิก"** → ออกจากโหมดแก้ไข

#### 5.5 เพิ่มตัวชี้วัด (ปุ่มเพิ่ม)
1. กดปุ่ม **"เพิ่ม"**
2. เลือก **ปีงบประมาณ**
3. (Admin) เลือก **อำเภอ** + **หน่วยบริการ** + **หน่วยงาน**
4. กดปุ่ม **"โหลดรายการ"** หรือรายการจะโหลดอัตโนมัติ
5. กรอกข้อมูลเป้าหมายและคะแนนรายเดือน
6. กดปุ่ม **"บันทึก"**

#### 5.6 กราฟแนวโน้ม (ปุ่ม 📈)
- กราฟเส้นผลงานรายเดือน (สีเขียว เส้นทึบ)
- เส้นเป้าหมาย (สีแดง เส้นปะ) — ถ้าเป้าหมายเป็นตัวเลข
- ถ้าเป้าหมายเป็นข้อความ → แสดง label สีม่วง
- ถ้าคะแนนเป็นข้อความ → แสดง annotation label บนกราฟ

#### 5.7 แบบฟอร์มบันทึกข้อมูล (Dynamic Form)
1. กดไอคอน 📋 ที่ตัวชี้วัดที่มีแบบฟอร์ม
2. เลือก **ปีงบประมาณ** + **เดือน** (เดือนที่คีย์แล้วจะไม่แสดง)
3. กรอกข้อมูลตามฟิลด์ที่กำหนด
4. กดปุ่ม **"บันทึกข้อมูล"**
5. Tab **"ข้อมูลที่บันทึก"** → ดู/แก้ไข/ลบข้อมูลที่บันทึกไว้

## 11. หน้าสร้างแบบฟอร์ม KPI

> **ใครเข้าถึงได้:** super_admin เท่านั้น

### ขั้นตอนการสร้างแบบฟอร์ม
1. เข้าเมนู **"สร้างแบบฟอร์ม KPI"**
2. เลือก **ตัวชี้วัด** ที่ต้องการสร้างแบบฟอร์ม
3. กำหนด **ชื่อฟอร์ม** + **คำอธิบาย**
4. เพิ่ม **ฟิลด์ข้อมูล** ตามต้องการ (ลาก/เรียงลำดับได้)
5. กำหนด **ฟิลด์ที่ sync ค่าไปยังผลงาน** (actual_value_field) — ค่าจากฟิลด์นี้จะแสดงใน Dashboard
6. กดปุ่ม **"บันทึก"** → ระบบสร้างตาราง MySQL ชื่อ `form_{table_process}` อัตโนมัติ

### ประเภทฟิลด์
| ประเภท | คำอธิบาย |
|--------|---------|
| text | ข้อความสั้น |
| number | ตัวเลข |
| date | วันที่ |
| select | เลือกจากตัวเลือก (กำหนด options) |
| textarea | ข้อความยาว (หลายบรรทัด) |
| checkbox | เลือก/ไม่เลือก |

### การใช้แบบฟอร์มใน Dashboard
- ตัวชี้วัดที่มีแบบฟอร์มจะแสดงไอคอน 📋 ในตาราง Dashboard
- กดเพื่อเปิดแบบฟอร์มกรอกข้อมูลรายเดือน
- เดือนที่คีย์ไปแล้วจะ **ไม่แสดงใน dropdown** (ป้องกันซ้ำ)
- เมื่อล็อคการคีย์ → แบบฟอร์มจะ **ปิดตาม** (สำหรับ user ทั่วไป)
- ข้อมูลจากฟิลด์ที่กำหนดจะ **sync ไปยังคอลัมน์ผลงาน** ใน Dashboard อัตโนมัติ

### การแบ่งชื่อตารางในฐานข้อมูล
| ตาราง | ชื่อ | ใช้สำหรับ |
|-------|------|---------|
| ตารางฟอร์ม (เก็บข้อมูลรายเดือน) | `form_{table_process}` เช่น `form_kpi_tab_001` | เก็บข้อมูลที่กรอกจากแบบฟอร์ม |
| ตาราง Export (สรุปรายปีต่อ hospcode) | `{table_process}` เช่น `kpi_tab_001` | ส่งออกข้อมูลสรุป |

## 12. หน้า Export ข้อมูล KPI

> **ใครเข้าถึงได้:** super_admin เท่านั้น

### ขั้นตอนการ Export

2. เลือก **ปีงบประมาณ**
3. เลือก **ตัวชี้วัด** ที่ต้องการส่งออก (เลือกทั้งหมดหรือบางตัว)
4. กดปุ่ม **"ตรวจสอบ"** → ระบบเปรียบเทียบข้อมูลก่อนส่งออก
5. ดูสถานะ:
   - 🟢 **มีการเปลี่ยนแปลง** → พร้อมส่งออก
   - 🔵 **ข้อมูลเหมือนเดิม** → ไม่ต้องส่งออก
   - ⚪ **ไม่มีข้อมูล** → ยังไม่มีการบันทึก KPI
6. กดปุ่ม **"ส่งออก"** → สร้าง/อัปเดตตารางในฐานข้อมูล

### โครงสร้างตาราง Export

**แบบ 1: ตัวชี้วัดที่ไม่มีแบบฟอร์ม** (บันทึกจากตาราง Dashboard)
| คอลัมน์ | คำอธิบาย |
|---------|---------|
| hospcode | รหัสหน่วยบริการ |
| byear | ปีงบประมาณ |
| target | เป้าหมาย |
| result | ผลรวมทั้งปี |
| m10 - m09 | ผลงานรายเดือน (ต.ค. - ก.ย.) |

**แบบ 2: ตัวชี้วัดที่มีแบบฟอร์ม** (บันทึกจาก Dynamic Form)
| คอลัมน์ | คำอธิบาย |
|---------|---------|
| hospcode | รหัสหน่วยบริการ |
| byear | ปีงบประมาณ |
| target | เป้าหมาย |
| result | — |
| {field_1}, {field_2}, ... | ฟิลด์จากแบบฟอร์มที่สร้าง (ไม่มีคอลัมน์เดือน) |

### หมายเหตุ
- Export ใช้ **INSERT ... ON DUPLICATE KEY UPDATE** → อัปเดตเฉพาะข้อมูลที่เปลี่ยน
- ข้อมูลเดิมจะ **ไม่ถูกลบ** (เพิ่ม/แก้ไขเท่านั้น)
- ตาราง Export แยกจากตารางฟอร์ม → ไม่กระทบข้อมูลต้นฉบับ
