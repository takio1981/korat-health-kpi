พ.ค. 2569 — Foundation & Performance

ปูฐานคุณภาพระบบให้แข็งแรงก่อนต่อยอดฟีเจอร์ใหม่
Sprint 1
สัปดาห์ 1–2
Database Index Audit
เพิ่ม composite index ที่ kpi_results(indicator_id, year_bh, hospcode), kpi_summary, kpi_sub_results
Query ↓ 30–50%
สัปดาห์ 1–2
Swagger / OpenAPI Docs
สร้าง /khupskpi/api/docs รวบรวม endpoint ทั้งหมด ~150 endpoints
เอกสาร API พร้อมใช้
สัปดาห์ 3–4
Unit Test Setup
Jest backend + Karma frontend, เขียน test core: auth, role guard, performKpiExport, checkKpiChanges
Coverage ≥ 30%
สัปดาห์ 3–4
ESLint + Prettier + Husky
Pre-commit hook กัน code messy, format อัตโนมัติทุก commit
Standard เดียว
KPI: Query latency ↓ 30% · Swagger UI พร้อม · Test coverage ≥ 30% · CI lint ผ่าน 100%

คีย์ข้อมูลรอ sync

Web Push Notifications
VAPID key + service worker + ส่งแจ้งเตือนการอนุมัติ/ตีกลับ
< 2 วินาที
สัปดาห์ 3–4
Touch Gestures
Swipe ลบ/แก้ไข, pull-to-refresh ใน dashboard mobile
UX เป็นธรรมชาติ

ผู้บริหารต้องเห็นภาพรวม → กราฟ + รายงานหลากหลาย + Export ได้ทุกฟอร์แมต
Sprint 3
สัปดาห์ 1–2
Custom Dashboard Widgets

สัปดาห์ 1–2
Export Excel / PDF
เพิ่มปุ่ม "ส่งออก" ที่ chart, report, dashboard — ใช้ ExcelJS + jsPDF
นำไปทำรายงานต่อได้
